GDPR Audit App - Test Plan

Test on a real site with Cookiebot + GTM configured. After each scan, cross-check a few results manually in browser DevTools / GTM / Cookiebot admin to catch false positives.

1. Page scan (18 Auto checks)

  • Run "Scan site" on a configured site - all Auto checks get results
  • Inline results: each check row shows summary text + itemized error findings (not just element names)
  • Findings drawer on a failing check: status badge, findings list with severity icons, "How to fix" steps, doc links
  • Findings drawer on a passing check: green "No issues found"
  • Findings drawer on unchecked check: check guide appears as fallback
  • Scan summary badge above checks: correct total, basic/full breakdown
  • Re-scan: results update in place, not duplicated
  • Verify false positive: pick a check marked OK (e.g. E2 Google Fonts) - open DevTools Network, filter "fonts.googleapis" - confirm the scanner result matches reality
  • Verify false positive: pick a check marked Issue (e.g. B1 tracking outside GTM) - check the flagged script actually exists in page source

2. Cookiebot API checks (6 checks: C1-C5, G3)

  • Run as part of "Scan site" on a site with Cookiebot ID set - 6 Cookiebot checks get results
  • Findings drawer: remediation steps appear for each
  • Remove Cookiebot ID from the site, re-scan - these 6 show "Cookiebot ID required" warning, no false results
  • Verify: open Cookiebot admin cookie report - compare categories against C1-C5 results to confirm accuracy
  • Check C5 (unclassified cookies): if it reports none, verify in Cookiebot admin that no unclassified cookies exist

3. GTM API checks (6 checks: A3-A5, B2-B4)

  • Run individually via "Run check" button on a site with GTM configured
  • Results appear inline and in findings drawer with remediation steps
  • Remove GTM ID - verify warning appears instead of results
  • Verify A3 (Cookiebot trigger): open GTM, check the Cookiebot tag's trigger - does it match what the app reports?
  • Verify B3 (consent gating): pick a non-Google tag in GTM, check its consent settings match the app's finding
  • Run all 6 GTM checks on a site with no non-Google tags - B3/B4 should report OK or N/A, not false issues

4. AI checks (9 checks: F2, F4, F6, G2, G6, G7, I1, I2, I8)

  • Verify OpenRouter credits in Settings before running
  • Run "AI Analyze" - all 9 AI checks get results
  • Findings drawer shows AI analysis section + remediation steps
  • Purple "AI" badge on all 9 checks
  • Verify G2 (accept/reject equally prominent): visually check the consent banner yourself - does the AI result match what you see?
  • Verify I1 (privacy policy complete): open the privacy policy, spot-check 2-3 elements the AI flagged as missing/present
  • Run AI checks on a site with no forms - F2/F4/F6 should report N/A or note no forms found, not hallucinate issues
  • Run AI checks with 0 credits - verify clear error message, no partial results

5. Browser checks (8 checks: G4, G5, H6, H8, K1-K4)

  • All 8 show blue "Browser" badge
  • Check guide (info icon) on each: step-by-step instructions present and understandable
  • Findings drawer on a manually-set "issue": "How to fix" section with remediation steps
  • Set status to OK, issue, N/A on different browser checks - verify saves persist after reload
  • Add an internal note to a browser check - verify note saves and "Internal notes" badge count updates

6. Manual checks (22 checks: C6, D2, E6, G9, H1-H5, H7, I5-I7, J1-J9)

  • All 22 show grey "Manual" badge
  • Check guide present on each
  • Findings drawer: remediation steps appear (newly added)
  • Set status + add note on a manual check - verify persistence
  • Set a check to N/A with a note explaining why - verify both save

7. Tier filtering

  • Toggle Basic/Full audit - visible check count changes (34 basic, 69 full)
  • Issue badges: separate basic/full counts, both visible regardless of tier toggle
  • Switch to Basic: only basic-tier checks visible, full-tier checks hidden
  • Switch back to Full: all checks visible, no data lost
  • Set a full-tier check to "issue" while on Full view, switch to Basic - that issue badge still shows in the full count

8. Filter bar

  • Click each filter badge: OK, Issues (basic), Issues (full), N/A, Not checked, Internal notes
  • Check list filters correctly for each, active filter shows ring highlight
  • Click active filter again to deselect - all checks reappear
  • Combine: set tier to Basic + filter to Issues - only basic-tier issues show

9. Findings drawer - cross-type spot check

Pick one check from each type and verify the full drawer content.

  • Page-scan check (e.g. A1): status, summary, findings, remediation, doc links
  • Cookiebot check (e.g. C3): status, summary, findings, remediation
  • GTM check (e.g. B3): status, summary, findings, remediation, dev/legal note
  • AI check (e.g. G7): status, AI analysis, remediation
  • Browser check (e.g. G4): status, remediation with DevTools instructions
  • Manual check (e.g. J1): status, remediation with client-facing steps
  • Platform-specific steps show Webflow/HubSpot badges where applicable
  • "Needs developer or legal input" note appears where marked

10. Chatbot - typical queries

  • "What checks are there for cookies?" - returns relevant checks from categories C, G, H
  • "How do I fix check B3?" - returns guide AND remediation steps (not just the guide)
  • "What's the status of this site?" (while on a site page) - returns audit progress with issue count
  • "Which sites have the most issues?" - returns compliance overview across all sites
  • "What does check A5 do?" - returns description, why it matters, and how to verify
  • "How do I test G4?" - explains the manual browser testing steps from the guide

11. Chatbot - less typical queries

  • "What's the difference between basic and full audit?" - should explain tier system sensibly
  • "Do I need a DPIA?" - should reference J9 and list the triggers
  • "What about US visitors?" - should find K2, K4 and explain opt-out model + GPC
  • "Is Google Analytics allowed without consent?" - should reference C2, explain EU vs UK difference
  • "What DPAs do I need?" - should reference J1, J2, list common vendors
  • "Which checks can't be automated?" - should list manual and browser check types
  • "Explain consent mode v2" - should reference B5 and the technical guide
  • "What if a cookie is unclassified?" - should reference C5 with remediation steps
  • Ask about a nonexistent check (e.g. "what is check Z9?") - should say not found, suggest search
  • Ask a vague question ("help") - should offer useful starting points, not hallucinate

12. Legend

  • Open "How this page works" - all 6 badge types listed: Auto, Cookiebot, GTM API, AI, Browser, Manual
  • Scan button descriptions match actual behavior
  • Status icons section accurate (not checked, OK, issue, N/A)
  • Row icons: ClipboardList = "Findings & fixes", Info = "Check guide"

13. Report

  • Generate report on a site with mixed results (OK, issues, not checked, N/A)
  • Browser and manual check results included alongside automated results
  • Remediation steps included for checks with issues
  • Report version increments correctly on re-generation

14. Edge cases

  • Scan a site with no Cookiebot ID and no GTM ID - only page-scan runs, others show warnings
  • Scan with invalid/unreachable URL - clear error, no crash, no partial bad data saved
  • Open app with no sites - empty state, no errors
  • Delete a site that has audit data - confirm deletion works, no orphaned data errors
  • Rapid-click "Scan site" twice - doesn't create duplicate scan runs
  • Set all checks to OK, then re-scan with new issues - verify old OK statuses update to issue where scanner finds problems
  • Compliance dashboard reflects current statuses accurately (spot-check 2-3 sites)
